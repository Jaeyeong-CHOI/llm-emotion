# 연구 컨텍스트 관리 정책 (llm-emotion)

목표: 채팅 맥락 길이를 줄이되, 의사결정 근거와 진행 상태는 유지.

## 1) 정보 계층화
- **채팅 요약(핵심만):** 현재 진행률, 중요 인사이트, 블로커, 다음 액션만 공유
- **상태 로그 파일:** `ops/research_state.json`, `ops/cron_runtime_status.json`, `LIVE_STATUS.md`
- **영구 아카이브:** `paper/logs/brief_log.md` (주/월 단위 누적)
- **원자료:** `results/`, `refs/`, `docs/evidence-table.md`, `ops/` 하위 JSON/로그

## 2) 보고 템플릿 (자동/수동 동일)
매 보고는 아래 4개 항목만 기본으로 함.
1. 현재 진행: `모듈명 - 수치 - 변화`
2. 의미: 왜 이 변화가 연구적으로 중요한지 1줄
3. 위험/제약: 인지된 오차/오류/실패 1개
4. 다음 24h 액션: 1개

필요 시에만 부연.

## 3) 삭제/보존 규칙
- 매일: 세부 채팅 로그는 축약본으로만 저장
- 매주: `paper/logs/brief_log.md`에 요약 1회 추가
- 장기 의사결정은 메모리에 누적(예: `memory/YYYY-MM-DD.md`)

## 4) 실수 방지
- 오버로드를 막기 위해 장문 텍스트를 실험 로그로 넣지 말 것
- 핵심 증거/그림/표/파라미터는 파일로 유지하고, 채팅엔 지표만 전달

## 5) 현재 적용
- 한국어 보고 고정
- 크론 리포트는 `마일스톤/결과/블로커/커밋` 이벤트만 알림
- 문헌 스크리닝/실험 러너/재현성은 `LIVE_STATUS.md`를 기준점으로 본문에서만 확인
